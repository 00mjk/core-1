Getting started as a JCrypTool Developer

This page is intended for new JCrypTool developers who need to set up their development environment and their workspace for the first time. In order to get you started as soon and as easy as possible, please follow these instructions carefully.

Requirements

In order to develop for the JCrypTool Core or Plug-ins project you need

  • A Java SE Runtime Environment (or Java SE Development Kit) in version 1.6 or newer.
  • Eclipse Indigo (3.7.x) in the Eclipse Classic or Eclipse for RCP/Plug-in Developers edition (other editions are not suitable for plug-in development without additional extensions). The latest Eclipse version (maintenance release) is always recommended.
  • A Subversion plug-in: Subversive (recommended) or Subclipse.

Installation and first start

Install the Java Runtime, Eclipse and your desired Subversion plug-in. No other plug-ins are required for JCrypTool development.

Start Eclipse afterwards and create a new (empty) workspace for JCrypTool. Open the Preferences and select the page Workspace in the General section. Here you have to choose UTF-8 as Text file encoding in the Other group.

Obtaining the JCrypTool sources

JCrypTool consists of two repositories: JCrypTool Core and JCrypTool Plug-ins. These two projects are located in different repositories. Both offer anonymous read access.

In order to start your development, you need to check out the JCrypTool sources from the Subversion repositories. All JCrypTool Core projects are required. Those projects contain the JCrypTool platform as well as some basic plug-ins required by the JCrypTool platform. All JCrypTool Plug-ins projects are optional. Those projects contain the crypto plug-ins. We strongly recommend checking out both repositories and all contained projects.

Checking out the JCrypTool sources

  • JCrypTool Core https://jcryptool.svn.sourceforge.net/svnroot/jcryptool
  • JCrypTool Plug-ins https://jctplugins.svn.sourceforge.net/svnroot/jctplugins

More information on Subversion access to our sources is available here. For more information on how to access our repositories visit

  • JCrypTool Core
  • JCrypTool Plug-ins

The next step is to check out the actual projects from the repositories linked before. If you use Subversive as SVN plug-in, this is quite simple:

  1. Select both repositories in the SVN Repositories view.
  2. Select Find/Check Out As... from the context menu.
  3. Finish the wizard on the first page, keeping all default settings (this will now take some time).
  4. A new dialog named Check Out Project shows up. Keep all defaults again and click Finish (this will again take some time).
  5. All plug-ins should appear as projects in your workspace.

Important: You must check out the trunk folder of every project! If you try to check out the projects directly, JCrypTool won't start! Do not close projects, all of them are required. A yellow exclamation mark in a feature project (these projects normally end with .feature) indicates normally that one or more (required) plug-ins are missing. In this case open the Feature Manifest Editor, switch to the Plug-ins tab and make sure that all listed plug-ins are available in your workspace. At this point ignore any warning in the feature projects org.jcryptool.core.dependencies.feature and org.jcryptool.core.dependencies.nl_de.feature.

Starting JCrypTool in your Eclipse environment (IDE)

After checking out at least all JCrypTool Core projects into your workspace, open the file jcryptool.product located in the org.jcryptool.core project with the Product Configuration Editor. Simply click on the Launch an Eclipse application link in the lower left on the first page. This starts JCrypTool and creates a run menu entry. Later launches can be done directly via the run menu entry or via the Launch an Eclipse application link in the Product Configuration Editor.

In case the first launch stops with an error message open the generated run menu entry (normally named jcryptool.product) and switch to the Plug-ins tab in the Run Configurations dialog. Click on the Add Required Plug-ins button and after that on the Run button. JCrypTool should be launched without any error message now.

JCrypTool will be launched in the language defined in your operating systems regional settings. Follow the instructions provided here to switch the active language.

Important: From time to time, new plug-ins or features will be added to the default JCrypTool configuration. After checking out those projects and/ or updating the corresponding features, you must use the Launch an Eclipse application link again to update your run entry configuration (otherwise the projects will be available in your workspace but won't be available during JCrypTool runtime).
