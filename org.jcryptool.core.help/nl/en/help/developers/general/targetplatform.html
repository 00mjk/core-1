Target Platform

A target platform decouples the target platform from the development platform. On one side this requires some additional setup work but on the other side makes later development much easier.

There are two possibilities to create a target platform: The automatic setup (recommended) and the manual setup.

Automatic Setup

Be aware that an active Internet connection is required and you will be downloading some megabytes into your workspace.

In order to use the target definition file, you'll have to check out our sources first. The JCrypTool target definition is contained in the file jcryptool.target in the org.jcryptool.releng project. Simply open this file in the Target Editor. Eclipse will now try to resolve all plug-ins referenced in this file, which will take quite a while depending on your hardware and your Internet connection speed. After resolving all plug-ins, the target definition editor will show something like the following screenshot:

The next thing to do is to click on the Set as Target Platform link in the upper right which will download and install all configured plug-ins as your active target platform. This includes the optional German translations from Eclipse Babel, so you can launch an English and a German JCrypTool right away. The setup process is finished after all JCrypTool projects have been automatically re-builded.

Manual Setup

The contained plug-ins in the Eclipse RCP SDK form the basis of the JCrypTool RCP, but do not contain all required plug-ins. This forces you to copy additional plug-ins manually from the normal Eclipse IDE into target platform directory.

Proceeding

  1. Download the Eclipse RCP SDK (http://download.eclipse.org/eclipse/downloads) (e.g. org.eclipse.rcp.source-3.6.1.zip) and extract it into an empty directory.
  2. Launch your normal Eclipse IDE installation, the Eclipse RCP SDK extracted before is not executable.
  3. Setup this directory in the JCrypTool workspace preferences as new target definition (switch to the Target Platform page, click the Add button for a new target platform and select the previously created directory as Location).
  4. After closing the preferences select the org.jcryptool.core.dependencies.feature project and open the feature.xml file. Switch to the Plug-ins tab and identify the missing plug-ins with the help of the yellow exclamation marks. Copy those jar files from the Eclipse IDE directory to your target platform. Remember to refresh your target definition in the preferences after adding new plug-ins. Repeat this step until all warnings are gone. Be careful: The dependencies feature does reference plug-ins and three included features. So open the included feature definitions as well (double click on each entry) and make sure the feature as well as all referenced plug-ins are contained in the target platform! You can only ignore the org.eclipse.rcp feature since this feature and all its plug-ins are provided by the RCP SDK installed before.
  5. Only plug-ins in the org.jcryptool.core.dependencies.nl_de.feature feature are optional, all other plug-ins are required.

This created a fully functional target platform, but only for your active operating system. This is good enough for JCrypTool. But in order to support other operating systems you need to execute the following steps as well:

  1. Download the Eclipse Delta Pack (http://download.eclipse.org/eclipse/downloads) (eclipse-3.6.1-delta-pack.zip for any platform) and extract it into the target directory created above. Overwrite all files if prompted.
  2. Refresh the target platform in the preferences.

This enables you to export JCrypTool for any supported platform. In order to get a completely German GUI you need to execute the following steps as well:

  1. Load Eclipse Babel (http://www.eclipse.org/babel) and download the following files for Eclipse Helios:
    1. BabelLanguagePack-eclipse-de_3.6.0.v20100814043401.zip
    2. BabelLanguagePack-rt.equinox.p2-de_3.6.0.v20100814043401.zip
  2. You can extract those files into the target platform created above. However we do recommend creating a different directory outside the rest of the target platform. Extract the two zip archives in there.
  3. Add a new Location in the target definition created above and select the directory containing the Language Packs.
  4. After the automatic rebuild the feature.xml in the org.jcryptool.core.dependencies.nl_de.feature project should not show any more warnings.

That is all you have to do for target platform setup. Every once in a while a new plug-in dependency is added to JCrypTool in the dependencies feature. In case the feature.xml displays a warning you have to update your target platform with one or more new plug-ins as described above.

We always develop against the latest stable Eclipse version. This requires to update the target platform after a new Eclipse release (e.g. Eclipse 3.6.2) is available as well. We recommend to start with a fresh target platform in this case and not to update the existing one.
