Tips and Tricks

Using JCrypTool in a network environment or on a USB stick

In principle it is possible to use JCrypTool in a network environment or on an external storage medium like a USB stick or a USB hard disk. To enable this you have to do a minor change in one of the configuration files. It is necessary to change the path to the default workspace folder. This folder is used by all plug-ins to store your specific preferences.

Windows/ Linux: Open the file JCrypTool.ini in your installation folder and search for the line -data @user.home/.jcryptool. Replace @user.home/.jcryptool with the absolute path to a folder available in your environment.

Mac OS X: The required changes are identical with the ones described above for Windows/ Linux. However the file JCrypTool.ini is located at JCrypTool.app\Contents\MacOS in your installation folder.

Switching the JCrypTool language

The initial JCrypTool language is based on your operating systems regional settings. An English Windows normally launches an English JCrypTool. However it is still possible that some plug-ins are shown only in German. The reason for that is simple: the plug-in is not available in English.

In order to change the JCrypTool language, open the Preferences and switch to the General page. Simply select the desired language. JCrypTool must be restarted in order for the changes to take effect.

Updating JCrypTool

JCrypTool contains an integrated update mechanism. Use the Help menu for all update operations. This menu contains the entries Check for Updates and Install new Software. However JCrypTool does not automatically search for new updates. You can either call the update search manually via the menu entry or you can change the Preferences on the Automatic Updates page to search automatically.

Updates are not visible

It is possible that even after a successful installation the updates/ changes are not visible. This may occur for newly installed plug-ins as well as updates for existing ones. The reason for that is the JCrypTool workspace folder, which caches and stores all plug-in information. Sometimes this information is not updated and the data stored there does not reflect the current installation any more.

To solve this you can either delete the .jcryptool folder in your document directory (inside your user home directory) manually or you can tell JCrypTool to do that job. Remember when trying to delete this folder that it is hidden on Mac OS X and Linux. To let JCrypTool delete this folder simply add the parameter -clean add the end when launching JCrypTool (e.g. add it to the shortcut on Windows). This automatically deletes the workspace folder. This folder is automatically recreated and initialized when starting JCrypTool the next time. Remember to remove the parameter afterwards.
