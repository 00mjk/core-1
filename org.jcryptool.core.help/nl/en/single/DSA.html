DSA

Description

DSA (Digital Signature Algorithm) is a US Federal Information Processing Standard (FIPS) for digital signatures, based on the mathematical concept of modular exponentiation and the discrete logarithm problem. DSA is a variant of the Schnorr and ElGamal signature scheme.

NIST (National Institute of Standards and Technology) proposed DSA for use in their Digital Signature Standard (DSS) in 1991, and adopted it as FIPS 186. The current specification is FIPS 186-4 from July 2013. Besides DSA, DSS also contains the signature schemes RSA und ECDSA.

FIPS 186-4 specifies to choose an approved cryptographic hash function H like SHA-2, and to choose for the modulus N one of the values 1024, 2048, 2048, or 3072.

Remark: Besides other hash functions, SHA-2 is offered in the JCT plugin Visuals → Hash Sensitivity.

Operation of the dialog

 

You have the choice to either create a DSA signature (sign) or to verify a DSA signature. Both, the signing and the verifying process here always uses the content of the current active editor as data (message).

If you select the radio button Sign, the private key of the signer is used and you have to specify the filename and path where to save the file with the signature. After these preparation steps click on the button Finish. Then you are asked for the password of the private key (the password is "1234" when using Alice's key from the keystore), and the corresponding signature is stored in a file.
Remark: In the opposite to the behavior with HMAC, the output is not stored in a new hex editor window in the middle of JCT but in a file.

If you select the radio button Verify, the public key of the signer is used and you have to specify the filename and path where to load the signature from. After these preparation steps click on the button Finish. Then a validation dialog pops up which shows the result of the verification (valid or invalid).

The signer's keys are always taken from the keystore. For verification an existing key from the keystore has to be chosen. For signing you can either select an existing asymmetric key (JCT always suggests a matching key from Alice if it exists) or you can first create a new one, store it in the keystore, and then select the newly generated key.

Pressing Create a new key for DSA in the keystore opens a new dialog called "New key pair" (e.g. for the user "Alice Whitehead" or better for yourself, by just writing a user name in the user field and overwrite "Alice").
