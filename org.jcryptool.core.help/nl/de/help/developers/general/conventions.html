Konventionen

JCrypTool besteht aus sehr vielen unterschiedlichen Plug-ins, entwickelt von vielen verschiedenen Personen. Daraus folgt zwangsläufig, dass die Plug-ins gewisse (qualitative) Unterschiede aufweisen. Um diese Unterschiede zu minimieren, geben wir eine Reihe von Konventionen und Empfehlungen vor, an die sich alle Entwickler halten sollten.

Workspace

Die wohl wichtigste Einstellung überhaupt betrifft Ihren Workspace. Da JCrypTool auf verschiedenen Plattformen - Linux, Mac OS X und Windows - läuft und entwickelt wird, ist hier eine plattformübergreifende Einstellung notwendig. Nach dem Anlegen des JCrypTool Workspaces sollten Sie daher als erstes diese Einstellung ändern. Öffnen Sie dazu in den Preferences unter General die Seite Workspace. Hier müssen Sie bei Text file encoding unter Other den Eintrag UTF-8 auswählen. Ohne diese Einstellung werden deutsche Umlaute und andere Sonderzeichen auf anderen Betriebssystemen nicht korrekt angezeigt.

Plug-in Kategorien

In der Regel werden die Krypto Plug-ins von interessierten Entwicklern beigesteuert. Die Namen der offiziellen JCrypTool Plug-ins beginnen dabei grundsätzlich mit org.jcryptool. Die folgende Tabelle zeigt die vollständigen Namen aller derzeit vorhandenen Kategorien.

Typ Bezeichnung
Analysen org.jcryptool.analysis
Klassische Algorithmen org.jcryptool.crypto.classic
Moderne Algorithmen org.jcryptool.crypto.modern
FlexiProvider Algorithmen org.jcryptool.crypto.flexiprovider
Spiele org.jcryptool.games
Visualisierungen org.jcryptool.visual

Der Name der Kategorie findet sich immer auch im so genannten Branding Plug-in wieder. Dieses Plug-in sorgt unter anderem für das Branding dieser Plug-ins, kann aber auch allgemeinen Code, der von mehreren Plug-ins dieser Kategorie verwendet wird, enthalten. Das Feature einer Kategorie erhält man, in dem man zum Branding Plug-in .feature hinzufügt.

Namenskonventionen

Unter Beachtung der Kategorie sollten Plug-ins als Namensende den Namen des implementierten Algorithmus oder Verfahrens enthalten, wie z.B. org.jcryptool.crypto.classic.xor oder org.jcryptool.visual.ecdh.

Implementierung

System.out.println() (oder deren Gegenstück System.err.println) sowie printStackTrace() Aufrufe haben in Plug-ins nichts zu suchen. Ebensowenig wie public static void main Methoden oder Aufrufe von System.exit(). Zum Loggen muss stattdessen das JCrypTool Logging Plug-in verwendet werden. Start- und Stop-Methoden sind nicht notwendig, der Lebenszyklus von Plug-ins wird von Eclipse selbst verwaltet.

Weitere Konventionen

Neben diesen Konventionen gelten ansonsten die Code Conventions for Java und die Eclipse Development Conventions and Guidelines.
