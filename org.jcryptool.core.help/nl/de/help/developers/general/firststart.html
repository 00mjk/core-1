Erste Schritte als JCrypTool Entwickler

Diese Seite gibt Ihnen einen ersten Eindruck von der Plug-in Entwicklung für JCrypTool. Um Ihre ersten Schritte so einfach wie möglich zu machen sollten Sie diese Informationen aufmerksam lesen und beachten.

Voraussetzungen

Um für das JCrypTool Core oder Plug-ins Projekt zu entwickeln benötigen Sie

  • Ein Java SE Runtime Environment (oder Java SE Development Kit) in Version 1.6 oder neuer.
  • Eclipse Indigo (3.7.x) in der Eclipse Classic oder Eclipse for RCP/Plug-in Developers Edition. Andere Eclipse Pakete enthalten ohne nachträgliche Erweiterungen keine Plug-in Entwicklungsumgebung. Wir empfehlen stets die Verwendung der neuesten Patch-Version.
  • Das EGit Plug-in: EGit, das Sie ganz einfach über die Indigo Update Site installieren können.

Installation und erster Start

Installieren Sie die Java Runtime, Eclipse und anschließend das EGit Plug-in. Weitere Plug-ins sind zur Entwicklung nicht zwingend erforderlich.

Starten Sie anschließend Eclipse und legen Sie einen neuen (leeren) Workspace für JCrypTool an. Öffnen Sie dann in den Preferences unter General die Seite Workspace. Hier müssen Sie bei Text file encoding unter Other den Eintrag UTF-8 auswählen. Ohne diese Einstellung werden deutsche Umlaute und andere Sonderzeichen auf anderen Betriebssystemen nicht korrekt angezeigt.

Zugriff auf die JCrypTool Sourcen

JCrypTool besteht aus zwei Git Repositories auf GitHub: Core und Crypto. Auf beide kann per anonymen Benutzer lesend zugegriffen werden.

Sofern Sie nur an der Krypto Plug-in Entwicklung interessiert sind können Sie Ihr Plug-in auch ohne das Auschecken der JCrypTool Sourcen entwickeln. Laden Sie sich einfach eine Binärvariante für Ihr Betriebssystem von JCrypTool herunter. Entwickeln Sie Ihr Plug-in ganz normal in Eclipse. Für die JCrypTool Integration (bzw. den Test Ihres Plug-ins) kopieren Sie Ihr neues Plug-in entweder in das JCrypTool dropins Verzeichnis und starten JCrypTool wie gewohnt. Auf diese Weise sparen Sie sich das im Folgenden beschriebene umfangreiche Setup, verwenden zur Entwicklung allerdings veraltete Sourcen (ein JCrypTool Release erscheint alle paar Monate, die Sourcen ändern sich teilweise aber wöchentlich oder gar täglich).

Um mit der Entwicklung zu beginnen empfehlen wir das Auschecken unserer Sourcen. Alle Plug-ins aus dem JCrypTool Core Repository sind notwendig. Diese Projekte enthalten die Basis JCrypTool Plattform und einige allgemein notwendige Plug-ins. Die Projekte im JCrypTool Crypto Repository sind optional. Wir empfehlen das Auschecken aller Plug-ins aus beiden Git Repositories.

Auschecken der JCrypTool Sourcen

  • JCrypTool Core https://github.com/jcryptool/core
  • JCrypTool Crypto https://github.com/jcryptool/crypto

Weitere Informationen rund um den Git Zugriff finden Sie auf den entsprechenden Repository Seiten.

Ein gelbes Ausrufezeichen in einem Feature Projekt (diese Projekte enden auf den Namen .feature) zeigt normalerweise an, dass ein oder mehrere notwendige Plug-ins fehlen. In diesem Fall öffnen Sie den Feature Manifest Editor, wechseln auf das Plug-ins Tab und überprüfen, welches Plug-in in Ihrem Workspace nicht vorhanden ist. Anschließend checken Sie auch diese(s) Plug-in aus dem jeweiligen JCrypTool Repository aus. Erscheint diese Warnung im Dependency Feature (org.jcryptool.core.dependencies.feature) fehlt ein notwendiges Eclipse Plug-in. In diesem Fall müssen Sie Ihre Eclipse Installation erweitern oder eine Target Platform einrichten. Lediglich Plug-ins im org.jcryptool.core.dependencies.nl_de.feature sind optional. Ohne diese Plug-ins (bzw. Fragmente) werden die original Eclipse Dialoge (wie z.B. die Benutzereinstellungen) nur auf Englisch verfügbar sein.

Starten von JCrypTool aus der Entwicklungsumgebung (IDE)

Nach dem erfolgreichen Auschecken aller JCrypTool Plug-ins öffnen Sie die Datei jcryptool.product im org.jcryptool.core Projekt mit dem Product Configuration Editor. Auf der ersten Seite dieses Editors finden Sie unten links den Link Launch an Eclipse application. Damit starten Sie JCrypTool. Gleichzeitig wird automatisch ein Eintrag im Run Menü generiert. Spätere Starts können Sie über diesen Link oder über den neuen Eintrag im Run Menü durchführen.

Sofern der JCrypTool Start mit einer Fehlermeldung abgebrochen wird öffnen Sie den generierten Run Menü Eintrag im Run Configurations Dialog und wechseln auf den Tab Plug-ins. Klicken Sie hier auf den Button Add Required Plug-ins und anschließend auf den Run Button. Jetzt sollte JCrypTool ohne Fehlermeldung starten.

JCrypTool ermittelt die Sprache basierend auf den Regionaleinstellungen Ihres Betriebssystems. Die Anleitung hier zeigt Ihnen, wie Sie die Sprache wechseln können (in einer Runtime funktioniert der Sprachwechsel über die JCrypTool Preferences nicht).

Wichtig: Von Zeit zu Zeit wird die JCrypTool Standardkonfiguration um weitere Plug-ins oder Features erweitert. Nachdem Sie diese neuen Projekte ausgecheckt haben müssen Sie JCrypTool einmal über den oben beschriebenen Link im Product Configuration Editor starten. Nur auf diese Weise wird Ihre Konfiguration aktualisiert. Andernfalls sind die neuen Plug-ins zwar in Ihrem Workspace verfügbar, in der JCrypTool Runtime aber dennoch nicht enthalten.
