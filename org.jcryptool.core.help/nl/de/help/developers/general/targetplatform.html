Target Platform

Eine Target Platform entkoppelt die Zielplattform von der Entwicklungsplattform. Dadurch ist zwar initial ein gewisser Mehraufwand notwendig, die spätere Entwicklung wird aber deutlich erleichtert.

Es gibt zwei Möglichkeiten, eine Target Platform aufzusetzen: Die automatische Installation (empfohlen) und die manuelle Installation.

Automatische Installation

Bitte beachten Sie, dass Sie für die automatische Installation eine aktive Internetverbindung benötigen und einige Megabytes in Ihren Workspace heruntergeladen werden.

Um die JCrypTool Target Definition Datei verwenden zu können, müssen Sie zunächst unsere Sourcen herunterladen. Die JCrypTool Target Definition befindet sich in der Datei jcryptool.target im org.jcryptool.releng Projekt. Öffnen Sie diese Datei einfach im Target Editor. Eclipse wird nun versuchen, die in dieser Datei referenzierten Plug-ins aufzulösen. Dieser Vorgang kann einige Zeit dauern, abhängig von der Geschwindigkeit Ihrer Internetverbindung. Nachdem er abgeschlossen ist sollte der Target Definition Editor in etwa wie im folgenden Screenshot aussehen:

Als nächstes müssen Sie auf den Set as Target Platform rechts oben klicken. Damit wird der Download und die Installation aller referenzierten Plug-ins gestartet und die neue Target Platform auf aktiv gesetzt. Enthalten sind hier auch die deutschen Übersetzungen von Eclipse Babel. Sie können JCrypTool anschließend also sowohl auf Englisch als auch auf Deutsch ausführen. Die automatische Installation ist abgeschlossen, nachdem JCrypTool mit der neuen Target Platform automatisch neu gebaut wurde.

Manuelle Installation

Die im Eclipse RCP SDK enthaltenen Plug-ins bilden die Basis für die JCrypTool RCP, enthält aber längst nicht alle notwendigen Plug-ins. Daher müssen verschiedene weitere Plug-ins von Hand aus der normalen Eclipse IDE ins Target Platform Verzeichnis kopiert werden.

Vorgehen

  1. Das Eclipse RCP SDK (http://download.eclipse.org/eclipse/downloads) herunterladen (z.B. org.eclipse.rcp.source-3.7.1.zip) und in ein leeres Verzeichnis entpacken.
  2. Starten Sie Ihre normale Eclipse IDE Installation, das zuvor entpackte Eclipse RCP SDK ist nicht ausführbar.
  3. Dieses Verzeichnis im JCrypTool Workspace in den Preferences als neue Target Definition einrichten (d.h. auf der Seite Target Platform per Add Button eine neue Target Platform anlegen und das im vorherigen Schritt angelegte Verzeichnis als Location auswählen).
  4. Nach dem Schließen der Preferences im org.jcryptool.core.dependencies.feature Projekt die Datei feature.xml öffnen. Auf dem Plug-ins Tab die in der Target Platform noch fehlenden Plug-ins anhand der gelben Warnung identifizieren und diese jar-Dateien aus dem Eclipse IDE Verzeichnis in die Target Platform kopieren. Nach dem Kopieren müssen Sie in den Preferences einen Refresh auf die Target Platform durchführen. Wiederholen Sie diesen Schritt so lange bis alle Warnungen verschwunden sind. Achtung: Das Dependencies Feature enthält neben den Plug-ins auch drei weitere Features (wovon Sie org.eclipse.rcp ignorieren können, dieses Feature wird von der RCP SDK zur Verfügung gestellt). Führen Sie einen Doppelklick auf jede Feature Referenz durch um dieses zu öffnen. Stellen Sie sicher, dass sowohl das Feature als auch dessen referenzierte Plug-ins in der Target Platform zur Verfügung stehen.
  5. Lediglich die Plug-ins aus dem org.jcryptool.core.dependencies.nl_de.feature Feature sind optional, alle anderen Eclipse Plug-ins müssen vorhanden sein.

Hiermit ist eine funktionsfähige Target Platform eingerichtet, allerdings nur für das aktuelle Betriebssystem. JCrypTool ist damit prinzipiell lauffähig. Um mehrere Betriebssysteme zu unterstützen müssen die folgenden Schritte zusätzlich ausgeführt werden:

  1. Den Eclipse Delta Pack (http://download.eclipse.org/eclipse/downloads) herunterladen (eclipse-3.7.1-delta-pack.zip ist für alle Plattformen identisch) und in das obige Verzeichnis entpacken. Vorhandene Dateien dabei überschreiben.
  2. Einen Refresh der Target Platform in den Preferences durchführen.

Jetzt kann JCrypTool für alle von Eclipse unterstützten Plattformen exportiert werden. Um eine vollständig deutsche Oberfläche zu erhalten sind noch einige weitere Schritte erforderlich:

  1. Von Eclipse Babel (http://www.eclipse.org/babel) die folgenden Dateien für Eclipse Indigo herunterladen:
    1. BabelLanguagePack-eclipse-de_3.7.0.v20111128043401.zip
    2. BabelLanguagePack-rt.equinox-de_3.7.0.v20111128043401.zip
    3. BabelLanguagePack-rt.equinox.p2-de_3.7.0.v20111128043401.zip
  2. Prinzipiell können diese Dateien in das oben erstellte Verzeichnis der Standard-Target-Platform entpackt werden. Empfehlenswerter ist es aber, die Sprachdateien vom Rest der Target Platform zu trennen. Dazu einfach ein neues Verzeichnis außerhalb der normalen Target Platform erstellen und die beiden zip-Archive hierin zu entpacken.
  3. In der oben angelegten Target Platform nun eine weitere Location hinzufügen und dabei das Verzeichnis mit den Language Packs auswählen.
  4. Nach dem automatischen Rebuild sollte die feature.xml aus dem org.jcryptool.core.dependencies.nl_de.feature Projekt keine Warnungen mehr anzeigen.

Damit ist das Setup der Target Platform komplett. Von Zeit zu Zeit kommen allerdings weitere Plug-in Abhängigkeiten zu JCrypTool hinzu, d.h. das Dependency Feature wird um weitere Einträge ergänzt. Sofern im feature.xml nach einem Update Warnungen angezeigt werden muss die Target Platform entsprechend aktualisiert werden.

Wir arbeiten stets gegen die aktuellste stabile Eclipse Version. Nach dem Erscheinen einer neuen Version (z.B. Eclipse 3.7.1) muss daher auch ein Update der Target Platform durchgeführt werden. Hier ist es empfehlenswert, mit einer neuen Target Platform zu starten.
