Tipps und Tricks

JCrypTool im Netzwerk oder auf einem USB-Stick verwenden

Prinzipiell kann JCrypTool auch im Netzwerk oder auf einem externen Speichermedium wie einem USB-Stick oder einer USB-Festplatte verwendet werden. Dazu müssen Sie jedoch eine Änderung in einer Konfigurationsdatei vornehmen. Konkret müssen Sie den Pfad zum Workspace-Ordner anpassen. In diesem Ordner speichern alle Plug-ins die von Ihnen getroffenen Einstellungen.

Windows/ Linux: Öffnen Sie die JCrypTool.ini in Ihrem Installationsordner und suchen Sie nach der Zeile -data @user.home/documents/.jcryptool. Hier müssen Sie @user.home/documents/.jcryptool durch einen in Ihrer Umgebung verfügbaren absoluten Pfad ersetzen.

Mac OS X: Die durchzuführende Änderung ist identisch wie unter Windows oder Linux. Allerdings befindet sich die JCrypTool.ini auf Mac OS X unter JCrypTool.app\Contents\MacOS in Ihrem Installationsordner.

JCrypTool aktualisieren

JCrypTool verfügt über einen integrierten Aktualisierungsmechanismus. Diesen können Sie über das Menü Hilfe aufrufen. Dort finden Sie die beiden Einträge Nach Aktualisierungen suchen und Neue Erweiterungen installieren.... JCrypTool sucht nicht automatisch regelmäßig nach Aktualisierungen. Sie müssen dies entweder manuell aufrufen oder in den Benutzervorgaben auf der Seite Automatische Aktualisierungen die entsprechenden Vorgaben festlegen.

Aktualisierungen werden nicht angezeigt

Es kann vorkommen, dass Aktualisierungen/ Änderungen trotz erfolgreicher Installation nicht angezeigt werden. Das kann bei neuen Plug-ins und auch bei Aktualisierungen von vorhandenen Plug-ins der Fall sein. Ursache hierfür ist der von JCrypTool verwendete Workspace Ordner, in dem die Informationen über alle installierten Plug-ins gespeichert werden. Bei Änderungen kann es nun passieren, dass die dort gespeicherten Einstellungen nicht aktualisiert werden.

In diesem Fall können Sie entweder das .jcryptool Verzeichnis in Ihrem Dokumente Verzeichnis (innerhalb Ihres Benutzerverzeichnisses) von Hand löschen. Unter Mac OS X und Linux ist dieser Ordner standardmäßig versteckt. Alternativ können Sie einmalig den Parameter -clean beim Aufruf von JCrypTool anhängen (z.B. über die Verknüpfung unter Windows). Hiermit wird der Workspace automatisch gelöscht und beim nächsten Start neu angelegt und initialisiert. Anschließend sollten Sie den Parameter wieder entfernen.
