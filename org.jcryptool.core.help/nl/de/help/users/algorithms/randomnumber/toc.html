xxxÜber MACs

xxxIn der Kryptographie ist ein Message Authentication Code (MAC) eine kurze Information, die zur Authentifizierung einer Nachricht verwendet wird – mit anderen Worten, um zu bestätigen, dass die Nachricht vom angegebenen Absender stammt (ihre Authentizität) und dass sie nicht verändert wurde (ihre Integrität). Verifizierer (die auch im Besitz des geheimen Schlüssels sind) können jegliche Änderung des Nachrichteninhalts erkennen.

xxxMAC-Funktionen ähneln zwar kryptographischen Hash-Funktionen, besitzen jedoch andere Sicherheitsanforderungen, und die so genannte Signierfunktion verwendet nicht nur die Nachricht, sondern auch einen symmetrischen Schlüssel als Eingabe. MACs unterscheiden sich auch von digitalen Signaturen, da MAC-Werte sowohl generiert als auch verifiziert werden, indem derselbe gemeinsame geheime Schlüssel verwendet wird, während digitale Signaturen den privaten Schlüssel des Absenders zum Signieren und den öffentlichen Schlüssel des Absenders zur Verifizierung verwenden.

Weitere Algorithmen für Zufallszahlen-Generatoren finden Sie in der Algorithmen-Perspektive von JCT. Siehe auch die Seite Allgemeine Informationen.

Weitere Informationen zu dem einzelnen Algorithmus finden Sie auf der entsprechenden Unterseite.

xxxÜber MICs und Message Digests

xxxDies ist nur eine Anmerkung über ähnliche Begriffe, die manchmal falsch verwendet werden.

xxxDer Begriff Message Integrity Code (MIC) wird häufig anstelle des Begriffs MAC verwendet, insbesondere in der Kommunikationstechnologie, um ihn von der Verwendung des MAC zu unterscheiden, d.h. der MAC-Adresse (für Media Access Control Address). MAC und MIC werden also austauschbar verwendet.

xxxManchmal wird jedoch MIC verwendet, um auf einen Message Digest zu verweisen, der sich von einem MAC unterscheidet, denn ein Message Digest verwendet keinen geheimen Schlüssel.
