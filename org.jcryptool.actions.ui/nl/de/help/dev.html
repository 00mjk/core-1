Aktionen

Die Aktionen Sicht bietet die Möglichkeit, Aktionen zu speichern und anschließend nochmals auszuführen. Damit Ihr Krypto Plug-in auch in dieser View verwendet werden kann sind die folgenden Punkte zusätzlich zu beachten.

Jeder Algorithmus hat eine zugehörige Action-Klasse, die von org.jcryptool.core.operations.algorithm.AbstractAlgorithmAction abgeleitet wird. In dieser Klasse ist eine abstrakte Methode public void run(IDataObject dataobject) definiert, die von der konkreten Action-Klasse implementiert werden muss.
Als Konfigurationsbasis dient ein DataObject, welches sämtliche Parameter aus dem Assistent aufnehmen muss. Die Aktionen-Sicht kann nur Parameter aufnehmen, die über das DataObject gehandelt werden.

Klassische Algorithmen verwenden ein ClassicDataObject, moderne Algorithmen ein ModernDataObject. Die Implementierung sieht in der Regel folgendermaßen aus, wobei normalerweise nur das DataObject durch den Typen des zum Algorithmus gehörenden DataObjects ersetzt werden muss.

Beispiel für die Implementierung eines klassischen Algorithmus

@Override
public void run(IDataObject dataobject) {
  AbstractClassicAlgorithm algorithm = new AdfgvxAlgorithm();

  ClassicDataObject d = (ClassicDataObject) dataobject;
  algorithm.init(d);

  super.finalizeRun(algorithm);
}

Beispiel für die Implementierung eines modernen Algorithmus

@Override
public void run(IDataObject dataobject) {
  DragonAlgorithm algorithm = new DragonAlgorithm();

  algorithm.dataObject = (SymmetricDataObject) dataobject;

  super.finalizeRun(algorithm);
}
